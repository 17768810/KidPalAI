# 四、软件架构（ESP32 + OpenClaw 云端）

### 1. 整体数据流（清晰易懂）

1. **ESP32 端**：

    - 本地离线唤醒（WakeNet）→ 检测“小书童小书童”

    - 唤醒后，麦克风采集语音 → 压缩 → 通过 WiFi 上传到**你的云服务器**

    - 接收云端返回的**文本回答** → 调用 TTS 转语音 → 喇叭播放

    - 控制彩灯、触摸、定时提醒

2. **云端 OpenClaw 服务**（你自己部署）：

    - 接收 ESP32 上传的语音 → 调用 ASR（百度/讯飞/OpenAI Whisper）转文字

    - 文字输入 OpenClaw → 大模型（GPT/通义千问/DeepSeek）理解意图

    - OpenClaw 执行：

        - 学习规划、任务管理、定时提醒（调用 OpenClaw 工具/技能）

        - 生成儿童化、正向的回答文本

    - 文本 → TTS（ElevenLabs/讯飞/百度）转语音 → 返回 ESP32

    - 存储学习记录、打卡数据、用户偏好

### 2. 关键技术选型

- **ESP32 固件**：ESP-IDF v5.5+ / MicroPython + PycoClaw（推荐，快速对接 OpenClaw）

- **本地唤醒**：乐鑫 ESP-SR（WakeNet + MultiNet），离线、低功耗

- **云端 ASR**：百度语音识别 / 讯飞听见 / OpenAI Whisper API

- **云端 TTS**：讯飞 TTS / 百度 TTS / ElevenLabs（儿童音色）

- **大模型**：通义千问 / DeepSeek / GPT-3.5-turbo（配置在 OpenClaw 中）

- **通信协议**：WebSocket / MQTT（ESP32 ↔ OpenClaw 服务器）
